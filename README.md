> Practice and implementation layer of the AIUZ Terra ecosystem: applications,
> DAO-like structures, prototypes, project surfaces, and operational bridges
> between FMP theory and deployable systems.

## Layer role

This repository is a practice-and-implementation layer.

It should hold:

- real applications and project structures;
- practical ecosystem integrations;
- implementation-facing documentation and release surfaces;
- a readable bridge from theory to practice.

It should not become:

- a symbolic shell with no practical entry;
- a donor governance repository;
- a theory-only archive;
- a generic dump of unrelated experiments.

## Reading path

1. `README.md`
2. `CITATION.cff`
3. implementation directories
4. `.github/workflows/`
5. linked practical repositories in the ecosystem

## Ecosystem position

- `FMP-CENTRAL-REPO` — central public hub
- `terra-legal` — donor governance and standards
- `AIUZ-Terra-codex` — broader ecosystem bridge
- `AIUZ` — practical node

## Operating rule

This repository must remain readable for humans first.

Its role is to expose practical implementation, not to collapse back into a
generic ProjectDNA placeholder.
